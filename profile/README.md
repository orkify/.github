# Orkify

Orkify brings a Vercel-like deployment workflow to self-hosted Node.js and Next.js apps.

It is built for teams that need more than PM2 offers, without taking on the operational overhead of Kubernetes. Orkify combines the control of running on your own infrastructure with a more modern, platform-like operations experience.

## What Orkify does

The free and open-source CLI helps you run apps on your own servers with:

- process management
- zero-downtime deployments
- log aggregation and log rotation
- orchestration across hosts
- basic MCP support for local development and remote apps
- no required accounts
- no tracking
- no external dependencies

The optional SaaS dashboard adds:

- real-time metrics
- error tracking with deobfuscated stack traces
- alerts and notifications
- rolling deployments
- built-in support chat
- Discord and Slack integrations
- team collaboration across hosts
- extended authenticated MCP support for teams

## Who it is for

Orkify is designed for developers and teams who:

- run Node.js or Next.js apps on their own infrastructure
- want something more modern and feature-rich than PM2
- do not want the overhead of Kubernetes
- want a workflow closer to modern cloud app platforms while keeping control of their own servers

## Platform support

The CLI runs natively on:

- Windows
- macOS
- Linux

It also includes a dedicated container run mode for Docker and container-based workflows.

## Projects

- **CLI** — open-source runtime, deployment, orchestration, and MCP tooling
- **Dashboard** — optional hosted SaaS for monitoring, alerts, support, and team workflows

## Philosophy

Orkify is built to keep self-hosting practical.

That means:

- simple local-first tooling
- open-source core
- optional hosted features
- predictable infrastructure choices
- no platform lock-in

Independent and bootstrapped.
